# The "Master" rules
# Project: SoSS (SaaS of SaaSs) - The Website Factory

## 🎯 Vision
A multi-tenant platform designed to build modular, professional, and multilingual sites. 
- **The Pilot:** Àgora (Charity/Social Work).
- **Core Concept:** A "Modular Engine" using Content Blocks rather than static templates.
- **Moats:** Dynamic AI Localization (8 languages:   locales: ['en', 'es', 'ca', 'eu', 'ga', 'fr', 'it', 'de']) and 4-state DNS matrix. The SoSS engine uses a single code route [domain] but shifts its behavior based on the tenant.category data.

## 💰 Business Model: Success-Based
- Free tier until revenue threshold $X is met.
- Platform Fee: ~1% "Success Fee" via Stripe Connect `application_fee_amount`.
- Paywalls: Tenants can lock specific Content Blocks behind their own subscriptions.

## 🏗️ Architecture: Feature-Sliced Design (FSD)
Strict "Gravity Rule": Communication only flows downwards. No upward or cross-slice imports within the same layer.

- **src/app/**: Next.js 16 Mandatory Folder. (Providers, Root Layout, Global CSS).
- **src/1-pages/**: Route views. Entry point for `app/[domain]/page.tsx`. Composed views. Grouped by scope: /soss, /admin, /tenant.
- **src/2-widgets/**: Composed chunks (e.g., `BlockRenderer`, `Navbar`). Grouped by scope: /soss, /admin, /tenant.
- **src/3-features/**: Interactive actions (e.g., `TranslateWithAI`, `StripeCheckout`). Grouped by scope: /admin (CMS actions), /tenant (Public interactions).
- **src/4-entities/**: Business logic/models (e.g., `Tenant`, `ContentBlock`). Business logic/models. Domain-first: tenant, block, user, organization.
- **src/5-shared/**: Infra/Foundation (e.g., `supabase-client`, `shadcn-ui`, `types`). Infra & UI Bricks. Divided by Segment: /ui (common/soss/tenant), /api, /lib, /config.

## 🛠️ Tech Stack & Standards
- **Framework:** Next.js 16.2 (Agent-Ready).
- **Database:** Neon (Serverless PostgreSQL) via Drizzle ORM, with strict RLS by `tenant_id`.
- **Auth:** Neon Auth (built on Better Auth), sessions and JWT stored directly in Neon DB.
- **UI:** shadcn/ui + Tailwind CSS.
- **Theming:** Use CSS Variables (HSL) for brand injection (e.g., `var(--primary)`).
- **SEO/GEO:** - Automate `hreflang` and JSON-LD (Schema.org) based on active blocks.
  - Optimize for Generative Engine Optimization (AI-search crawlers).

## 🤖 AI Worker Rules
- When generating UI, use **shadcn/ui** and Tailwind.
- Always check for **Multilingual** support (is the text being translated?).
- Always check for **Multi-tenancy** (is there a `tenant_id` filter?).
- Keep code **Generic**: Can this "Charity" block be used for a "Wedding" or "Business" tenant?
- **Zero Hardcoding:** All colors/fonts must use CSS variables (`--primary`).
- **RLS First:** Every Supabase query MUST include a `.eq('tenant_id', tenantId)` filter.
- **The Wedding Test:** Every block must be generic enough to serve different categories.
- **Public API:** Every slice MUST have an index.ts.
- **Namespaced Slices:** Place code in the correct sub-folder (SoSS vs Admin vs Tenant).
