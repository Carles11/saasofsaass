# The "Master" rules
# Project: SoSS (SaaS of SaaSs) - The Website Factory

## 🎯 Vision
A multi-tenant platform designed to build modular, professional, and multilingual sites. 
- **The Pilot:** Àgora (Charity/Social Work).
- **Core Concept:** A "Modular Engine" using Content Blocks rather than static templates. Templates are TypeScript appearance presets only (themeClass, containerClass) — no blocks, no content, no DB storage.
- **Moats:** Dynamic AI Localization (8 locales: 'en', 'es', 'ca', 'eu', 'ga', 'fr', 'it', 'de') and single-codebase multi-tenant DNS routing via Next.js proxy middleware.

## 💰 Business Model: Success-Based
- Free tier until revenue threshold $X is met.
- Platform Fee: ~1% "Success Fee" via Stripe Connect `application_fee_amount`.

## 🏗️ Architecture: Feature-Sliced Design (FSD)
Strict "Gravity Rule": Communication only flows downwards. No upward or cross-slice imports within the same layer.

- **src/app/**: Next.js 16.2 Mandatory Folder. (Providers, Root Layout, Global CSS).
- **src/1-pages/**: Route views. Composed views. Grouped by scope: dashboard, marketing, tenants.
- **src/2-widgets/**: Composed chunks (e.g., `BlockRenderer`, `Navbar`, `SiteBuilder`). Grouped by scope: dashboard, tenant.
- **src/3-features/**: Interactive actions (e.g., `manage-site-blocks`, `manage-entities`, `auto-translate-content`).
- **src/4-entities/**: Business logic/models (e.g., `Tenant`, `Block`, `TenantContent`). Domain-first: tenant, block.
- **src/5-shared/**: Infra/Foundation (e.g., `shadcn-ui`, `auth`, `db`, `types`, `config`, `store`). Divided by Segment: /ui, /lib, /config, /types.

## 🛠️ Tech Stack & Standards
- **Framework:** Next.js 16.2 (Turbopack).
- **Database:** Neon (Serverless PostgreSQL) via Drizzle ORM.
- **Auth:** Neon Auth (built on Better Auth), sessions via API proxy at `/api/auth/[...path]`.
- **UI:** shadcn/ui + Tailwind v4 + next-themes (dark/light).
- **Templates:** TypeScript source of truth (`config/templates.ts`). Text slug on tenant. Appearance-only presets. No DB storage, no UUID FK, no seed script.
- **Theming:** Use CSS Variables (HSL) via semantic shadcn vars for platform pages. Tenant blocks use template-provided themeClass.
- **SEO/GEO:** Automate `hreflang` and JSON-LD (Schema.org) based on active blocks. Optimize for Generative Engine Optimization (AI-search crawlers).

## 🤖 AI Worker Rules
- When generating UI, use **shadcn/ui** and Tailwind.
- Always check for **Multilingual** support (is the text being translated?).
- Always check for **Multi-tenancy** (is there a `tenant_id` filter?).
- **Zero Hardcoding:** Platform pages use semantic shadcn CSS vars. Tenant blocks use template themeClass.
- **Drizzle ORM:** Every query MUST filter by `tenant_id` where applicable.
- **Public API:** Every slice MUST have an index.ts.
- **Namespaced Slices:** Place code in the correct sub-folder (dashboard vs tenant).
