# DNS Routing & Proxy Middleware

## 🎯 Goal
Provide seamless multi-tenant experience where subdomains are rewritten to a single dynamic engine.

## 🗺️ DNS Matrix

| Type   | Host                         | Rewrite Target        |
|--------|------------------------------|-----------------------|
| Root   | saasofsaass.com / localhost:3000 | `/(marketing)`      |
| App    | app.saasofsaass.com / app.localhost:3000 | `/(dashboard)` |
| Tenant | *.saasofsaass.com / *.lvh.me | `/(tenants)/[domain]` |

## 🧠 Middleware Logic (`src/proxy.ts`)
next-intl runs first for locale handling → locale prefix stripped → DNS rewrite applied with intl headers forwarded.

File is named `proxy.ts` (not `middleware.ts`) — Next.js 16.2 convention.

## 🏗️ FSD Page Composition
- **Marketing:** `src/1-pages/marketing/` — handles `(marketing)` route group
- **Dashboard:** `src/1-pages/dashboard/` — handles `(dashboard)` route group
- **Tenant:** `src/1-pages/tenants/` — dynamically renders tenant blocks based on `domain` param
