# DNS matrix & Middleware logic.

SoSS Routing & DNS Architecture

🎯 Goal
To provide a seamless multi-tenant experience where subdomains and custom domains are rewritten to a single dynamic engine.

🧠 Middleware Logic
The middleware.ts acts as the primary gatekeeper:Identify hostname from headers.Exclude internal Next.js assets (_next, api, static).Rewrite based on the DNS Matrix above.

🏗️ FSD Page Composition
Marketing: Uses @/1-pages/soss/*
Dashboard: Uses @/1-pages/admin/*
Tenant: Uses @/1-pages/tenant/* (Dynamic injection based on tenant_id).
